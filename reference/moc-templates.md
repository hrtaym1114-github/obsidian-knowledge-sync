# MOC Templates

This reference file contains templates for auto-generating MOC (Map of Contents)
files in both Japanese and English, along with idempotency marker usage.

## Idempotency Markers

All auto-generated sections MUST be wrapped in these markers:

```
<!-- obsidian-knowledge-sync:start -->
(auto-generated content goes here)
<!-- obsidian-knowledge-sync:end -->
```

### Rules

1. The marker prefix `obsidian-knowledge-sync` is unique to this skill.
   Do NOT use generic markers like `<!-- auto-generated -->`.
2. Content between markers is replaced on every `--update` run.
3. Content OUTSIDE markers is never modified by this skill.
4. If an existing MOC file has no markers, the skill must NOT overwrite it.
   Instead, show a warning and suggest the user add markers manually.

### Marker Placement in Templates

```markdown
# _Category-MOC

(User's custom introduction - preserved on update)

<!-- obsidian-knowledge-sync:start -->
## Auto-generated note index
(this section is regenerated on each --update)
<!-- obsidian-knowledge-sync:end -->

(User's custom notes below - preserved on update)
```

## Japanese Template (--language ja)

Use this template when `--language ja` is specified (default).

```markdown

> このMOCは Obsidian Knowledge Sync により自動生成されました。
> 手動で追記した内容はマーカー外に記述すれば保持されます。

<!-- obsidian-knowledge-sync:start -->
## ノート一覧

**最終更新**: [YYYY-MM-DD]
**合計**: [N] 件

[IF hierarchical style AND subfolders exist]
### [Subfolder1 Name]
- [[Note-A]] - [first heading or filename] ([YYYY-MM-DD])
- [[Note-B]] - [first heading or filename] ([YYYY-MM-DD])

### [Subfolder2 Name]
- [[Note-C]] - [first heading or filename] ([YYYY-MM-DD])

[ELSE IF flat style OR no subfolders]
- [[Note-A]] - [first heading or filename] ([YYYY-MM-DD])
- [[Note-B]] - [first heading or filename] ([YYYY-MM-DD])
- [[Note-C]] - [first heading or filename] ([YYYY-MM-DD])
[END IF]

## 関連MOC
[List any other _*-MOC.md files found in parent or sibling folders]
- [[_Related-MOC]]
<!-- obsidian-knowledge-sync:end -->

#moc #[category-tag] #auto-generated
```

## Field Extraction Rules

### Note Title
Extract the display title for each note in this priority order:
1. First `# ` (H1) heading in the file -> use as title
2. YAML frontmatter `title:` field -> use as title
3. Filename without `.md` extension -> use as title

### Note Date
The date shown in parentheses `(YYYY-MM-DD)` is the file's last-modified date.
Obtain it via:
```bash
# macOS
stat -f "%Sm" -t "%Y-%m-%d" "filepath"
# Linux
stat -c "%y" "filepath" | cut -d' ' -f1
```

### Related MOC Discovery
To populate the "Related MOC" section:
```bash
# Find MOC files in parent directory
Glob: ${TARGET}/../_*-MOC.md
# Find MOC files in sibling directories
Glob: ${TARGET}/../*/_*-MOC.md
```

Exclude the current MOC file from the list.

## Update Behavior (--update)

### Adding New Files
When `--update` is specified:
1. Read the existing MOC file
2. Parse the content between `obsidian-knowledge-sync:start` and `end` markers
3. Extract the list of currently linked notes
4. Scan the target folder for all .md files
5. Identify NEW files (present in folder but not in MOC)
6. Regenerate the entire section between markers, including both old and new entries
7. Sort entries: by subfolder first (hierarchical), then alphabetically within each group

### Removing Deleted Files
If a file listed in the MOC no longer exists in the folder:
- Remove it from the regenerated section
- This happens automatically since the section is fully regenerated from the current folder contents

### Preserving User Content
Everything outside the markers is untouched. Examples of preserved content:
- Custom introduction paragraphs above the start marker
- Personal notes or annotations below the end marker
- Tags at the bottom of the file

## MOC File Naming Convention

| Target Folder | MOC Filename |
|--------------|-------------|
| `04_Memory/AI/` | `_AI-MOC.md` |
| `04_Memory/Technical/` | `_Technical-MOC.md` |
| `04_Memory/Business/` | `_Business-MOC.md` |
| `03_Input/Blog-Research/` | `_Blog-Research-MOC.md` |

Pattern: `_[BaseFolderName]-MOC.md`

- Always use underscore `_` prefix (Obsidian convention for index/MOC files)
- Use the exact folder basename (preserve casing)
- Always append `-MOC` suffix before `.md`
